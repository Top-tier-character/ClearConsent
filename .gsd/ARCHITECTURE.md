## Data Flow

1. User uploads PDF/image/text on `/analyze`
2. Client calls `/api/extract-pdf` or `/api/ocr` to get raw text
3. Client calls `/api/analyze` → Groq LLM → JSON analysis (risk score, flags, action plan, figures)
4. Result stored in Zustand store + Convex `document_analyses` table
5. User can chat with AI assistant (`/api/assistant`) for clarifications
6. User can run EMI simulation (`/api/simulate`)
7. History persisted in Convex `consent_records` and `document_analyses`
